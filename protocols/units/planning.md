# Unit: Planning (Architectural Specs)

- **Protocol P-1 (Logic-First TOC):** Before writing the document, prioritize designing a detailed Table of Contents (TOC) based on the logical steps required to achieve the final goal.
- **Protocol P-2 (Incremental Deep-Dive):** Write each item of the designed TOC sequentially using a multi-pass approach (e.g., Phase 1: High-level TOC, Phase 2: Detailed sub-TOC, Phase 3: Full content generation). Do not proceed to the next item until all details of the current section are fully complete.
- **Protocol P-3 (Chunking Strategy):** If the document is extensive, divide the work into sections to maintain context, and establish user confirmation checkpoints at each stage.
- **Protocol P-4 (No Omission Policy):** Strictly prohibit expressions like 'omitted' or 'skipped'. Complete all policies/designs with literal text so the document is fully implementable on its own.
- **Protocol P-5 (Workflow Validation):** Perform a final verification to ensure the completed details match the user workflows and logical steps defined in P-1.
