# Unit: Markdown (Documentation Integrity)
- **Protocol M-1 (Structural Hierarchy):** Use the H1 (#) heading only once, and clearly define the document's hierarchical structure using H2 (##) and below.
- **Protocol M-1.1 (Incremental Rendering):** For large documents, follow a 3-step rendering pipeline: Phase 1 (Create skeleton TOC using H2), Phase 2 (Define sub-structures and keywords using H3), Phase 3 (Flesh out all literal text and details for each H3 section). Do not generate the entire document at once.
- **Protocol M-2 (Literal Specification):** Absolutely prohibit expressions like 'omitted', 'skipped', or '...'. Complete all specifications with literal text.
- **Protocol M-3 (Data Visualization):** Must use tables to enhance the readability of complex data or comparative information.
- **Protocol M-4 (Content Consistency):** Maintain terminology consistency when referring to the same object throughout the entire document.
- **Protocol M-5 (Logic Modeling):** Utilize Mermaid or UML diagrams within code blocks when explaining complex flows or structures.
- **Protocol M-6 (Reference Integrity):** Verify the integrity of external links or asset references within the document and ensure correct paths are used.
